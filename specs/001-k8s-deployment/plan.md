# Implementation Plan: Kubernetes Deployment Package

**Branch**: `001-k8s-deployment` | **Date**: 2026-03-22 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/001-k8s-deployment/spec.md`

## Summary

Package PII Proxy as a Helm chart that deploys Envoy and the ext_proc PII-scrubbing
sidecar as a dual-container Pod in an existing Kubernetes 1.27+ cluster. A single
`values.yaml` controls all environment-specific settings (upstream URL, image
references, replica count, resource limits, gRPC port, spaCy model name). A
`ClusterIP` Service exposes Envoy in-cluster; an optional `LoadBalancer` Service
is toggled via values. Local integration testing uses **minikube** with
`minikube image load` for registry-free image delivery.

## Technical Context

**Language/Version**: Helm 3.x (YAML templates); Python 3.12 (ext_proc — existing)  
**Primary Dependencies**: Helm 3.x, kubectl 1.27+, envoyproxy/envoy:v1.33.0, minikube v1.32+ (local testing), spaCy `en_core_web_lg`, Presidio  
**Storage**: N/A — stateless proxy  
**Testing**: `helm lint`, `helm test` hook Pod (end-to-end PII scrubbing), minikube for local integration  
**Target Platform**: Kubernetes 1.27+ (CNCF-conformant); minikube for local dev/testing  
**Project Type**: Infrastructure / deployment package (Helm chart)  
**Performance Goals**: ≥50 req/s per ext_proc replica at 2 vCPU / 4 GiB RAM (SC-004)  
**Constraints**: ext_proc NotReady 2-3 min during spaCy load; `failure_mode_allow: false`; gRPC NOT exposed outside pod  
**Scale/Scope**: Single cluster, single namespace; horizontal scale via `replicaCount`

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

| Principle | Status | Notes |
|-----------|--------|-------|
| I. Privacy-First | ✅ PASS | `failure_mode_allow: false` preserved in Envoy ConfigMap template; Envoy returns 5xx on ext_proc failure — PII never forwarded raw (FR-008, SC-005) |
| II. Transparently Invisible | ✅ PASS | Sidecar pattern — Envoy+ext_proc in same Pod; gRPC on localhost; no upstream code changes required |
| III. Test-First | ✅ PASS | Helm test hook (`test-scrubbing.yaml`) defined in contracts before implementation; tests PII scrubbing end-to-end |
| IV. Simplicity & Focused Scope | ✅ PASS | Helm chart only (no Kustomize fallback); ext_proc unchanged; chart scope limited to packaging/deploying |
| V. Containerized & Reproducible | ✅ PASS | All runtime components containerized; minikube + `image load` enables reproducible local testing; config via `values.yaml` only |

**Security gate**: gRPC Service port 50051 is NOT exposed outside the pod (no Service selector for that port) — compliant with constitution Security Requirements. ✅

**Post-design re-check**: TCP socket readiness probe on port 50051 is valid because `grpc.server.start()` is called only after `PiiScrubber.__init__()` completes (model load). Port open = model ready. No code changes required.
